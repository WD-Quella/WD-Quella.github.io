    # 数据类型

    为了让程序有价值，我们需要能够处理最简单的数据单元：数字，字符串，结构体，布尔值等。 TypeScript支持与JavaScript几乎相同的数据类型，此外还提供了实用的枚举类型方便我们使用。

    # 布尔类型（boolean）

    和大多数语言一样布尔类型只有两个值true和false

    let isbool:boolean=true;
    1

    # 数字类型（number）

    number可以理解为整型和浮点型的合并支持 双精度 64 位浮点值。它可以用来表示整数和分数。

    let decLiteral: number = 6; //十进制 6
let hexLiteral: number = 0x0f; //十六进制 0f 即15
let binaryLiteral: number = 0b1010; //二进制 1010 即10
let octalLiteral: number = 0o011; //八进制 011 即9

    # 字符类型（string）

    字符类型的形式和javascript基本一样，用‘或者"以及```来标识。

    let name:string="robin";
    1

    ts中还支持模版语法,但是注意使用模版语法必须用多行```来标识。

    let mname:string="robin";
let myname:string=`my name is ${mname}`;

    # 数组类型（Array）

    常用数组声明有两种数组泛型的声明。

    let arr:number[]=[1,2,3];
let arr_1:Array<number> = [1, 2, 3];

    # 元组类型（Tuple）

    元组相当于指定各元素类型的数组,赋值和访问越界属性的时候会报错。

    let tuple:[string,number];
tuple=["str",3]

    # 枚举类型（enum）

    枚举类型声明后相当于一个变量集合，默认为0开头的序号集合，如果前面元素有赋值，则依次类推

    enum Color{red,blue,green}
let r:Color=Color.red;//r为0

enum Color1{red,blue=4,green}
let r1:Color1=Color1.green;//r为5
enum Color2{red,blue=4,green,orange}
let r2:Color2=Color2.orange;//r为6

    # Any类型（any）

    any相当于javascript的数据类型，即弱类型，根据赋值改变对应的数据类型。 any类型在不确定数据类型的时候很有用可以指定第三方变量或者数据。

    let anystr:any="str";
anysty=4;
anysty.toFixed();

    Void类型和表示任意类型的any相反，表示没有类型，通常用在函数没有返回值的时候声明。

    console.log("is void function");

    # Null 和 Undefined类型

    TypeScript里，undefined和null两者各自有自己的类型分别叫做undefined和null。 和 void相似，它们的本身的类型用处不是很大。

    let u: undefined = undefined;
let n: null = null;
    1
    2

    TIP

    --strictNullChecks标记 比如某个函数参数你想传入一个 string或null或undefined，此时联合声明string | null | undefined

    let s:string | null | undefined=null;
    1

    如果编译的时候对指定对string变量赋值null或者undefined报错那么说明--strictNullChecks是开启的 可以在tsconfig.json中配置

    {
        "compilerOptions": { //编译选项,可以被忽略，这时编译器会使用默认值
            "strictNullChecks": false, //在严格的null检查模式下，null和undefined值不包含在任何类型里，只允许赋值给void和本身对应的类型。
        }
}

    # Never 类型

    never类型表示的是那些永不存在的值的类型。 例如， never类型是那些总是会抛出异常或根本就不会有返回值的函数表达式或箭头函数表达式的返回值类型； 变量也可能是 never类型，当它们被永不为真的类型保护所约束时。 当然平常可以用any替代never。

    // 返回never的函数必须存在无法达到的终点
let error:never=((msg:string)=>{
    throw Error(msg);
})()

// 返回never的函数必须存在无法达到的终点
function infiniteLoop(): never {
    while (true) {
    }
}

    # Object类型

    object表示非原始类型，也就是除number，string，boolean，symbol，null或undefined之外的类型。

    declare function create(o: object | null): void;
create({ prop: 0 }); // OK
create(null); // OK
    1
    2
    3

    TIP

    declare关键字只在编译中声明有效，用以判断传入参数类型是否正确，多用于函数声明和模块声明

    # 类型断言

    ts的类型检查在不同类型变量之间操作时会造成麻烦这时就需要类型断言告诉ts编译。 所以类型断言接近于其他语言中的强制类型转换。

    let someValue: any = "this is a string";
let strLength: number = (<string>someValue).length;
//或
let someValue: any = "this is a string";
let strLength: number = (someValue as string).length;
