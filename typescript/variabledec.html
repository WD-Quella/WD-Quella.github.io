    # 变量声明

    ts变量声明同es6的变量声明均为在var的基础上新增let和const。而在理解变量之前先要知道声明是块作用域，最简单的理解就是{}包着的语句内容就是在一个块作用域底下。

    # var声明

    js基础变量声明方式，局部的变量如果找不到声明会从依次从外层作用域中寻找声明变量直到全局变量，而变量声明存在部分怪异之处。

    var c=1;
var c=2;
var c=3;
consle.log(c)
//输出3，var可以重复声明且只会得到唯一的最后声明的该变量。
function f(){
    a=12;
}
f();
console.log(a);//结果输出12
//同理
function f(){
    function f1(){
        b=10;
    }   
    a=12;
    console.log(b);//error b is not defined
    f1();
    console.log(b);//结果输出10
}
f();

    TIP

    我们传给setTimeout的每一个函数表达式实际上都引用了相同作用域里的同一个i。 setTimeout在若干毫秒后执行一个函数，并且是在for循环结束后。 for循环结束后，i的值为10。 所以当函数被调用的时候，它会打印出 10！

    上面这几种情况实际我们想要的输出是对应的123序号的返回而这也是js变量在for的块作用域下运行异步的操作（比如异步的ajax请求等）的时候的怪异之处。 解决方案可以运用上面所讲的闭包来解决。

    for (var i = 0; i < 10; i++) {
    (function(i){
        setTimeout(
        function() { 
            console.log(i); 
            },
         100 * i);
    })(i)
}
//或
for (var i = 0; i < 10; i++) {
    (function(){
        var tmp=i
        setTimeout(
        function() { 
            console.log(tmp); 
            },
         100 * tmp);
    })()
}

    # let声明

    let声明和var声明接近但是let声明只存在当前块作用域以及嵌套的内作用域，作用域外部获取不到let声明的变量。

    function f(input: boolean) {
    let a = 100;

    if (input) {
        // Still okay to reference 'a'
        let b = a + 1;
        return b;
    }

    // Error: 'b' doesn't exist here
    return b;
}

    # const声明

    const 和let 接近但是const声明后的变量不允许更改

    const numLivesForCat = 9;
const kitty = {
    name: "Aurora",
    numLives: numLivesForCat,
}

// Error
kitty = {
    name: "Danielle",
    numLives: numLivesForCat
};

// all "okay"
kitty.name = "Rory";
kitty.name = "Kitty";
kitty.name = "Cat";
kitty.numLives--;

    # 对象解构

    let s={
     a:1,
     b:"123",
     c:22
}
let {a,b}=s
console.log('b: ', b);//输出 123
console.log('a: ', a);//输出 1
let na,nb;
({a:na,b:nb}=s)
console.log('nb: ', nb);//输出 123
console.log('na: ', na);//输出 1
//默认值
function keepWholeObject(wholeObject: { a: string, b?: number }) {
    let { a, b = 1001 } = wholeObject;
}
function f({ a, b = 0 } = { a: "" }): void {
    // ...
}
f({ a: "yes" }); // ok, default b = 0
f(); // ok, default to {a: ""}, which then defaults b = 0
f({}); // error, 'a' is required if you supply an argument
